2. PHƯƠNG PHÁP LUẬN KIỂM THỬ

2.1. Test Design Methodology

Nguyên tắc: Mỗi Test Case kiểm tra 4-5 chức năng liên quan

Lý do áp dụng:

  • Hiệu quả cao: Một test case có thể phát hiện nhiều lỗi liên quan
  • Coverage tốt: Đảm bảo kiểm tra đầy đủ các luồng nghiệp vụ
  • Phát hiện lỗi tích hợp: Phát hiện lỗi khi các chức năng tương tác với nhau
  • Giảm redundancy: Tránh lặp lại các test case tương tự

Ví dụ Test Case Design:

TC-ORDER-FLOW-001: Quy trình đặt hàng hoàn chỉnh

Chức năng Mô tả Kỹ thuật kiểm thử
1. Browse & Filter Xem danh sách món và lọc theo danh mục Equivalence Partitioning, Boundary Value
2. Add to Cart Thêm món vào giỏ và cập nhật số lượng State Transition, Data Flow
3. Calculate Total Tính toán tổng tiền (món + phí giao hàng) Boundary Value Analysis, Error Guessing
4. Place Order Đặt hàng với thông tin giao hàng State Transition, Integration Testing
5. Payment & Confirm Thanh toán và xác nhận đơn hàng Integration Testing, Security Testing

2.2. Testing Techniques Applied

Kỹ thuật Mô tả Áp dụng cho Phương pháp luận

Grey Box Testing Kiểm thử với kiến thức một phần về cấu trúc 8 test cases Database Testing, API Testing, Integration Testing
White Box Testing Kiểm thử cấu trúc code và logic 4 test cases Code Coverage, Security Testing, Performance Testing
Integration Testing Kiểm thử tích hợp giữa các module 2 test cases End-to-End Testing, System Integration
Security Testing Kiểm thử bảo mật 2 test cases Penetration Testing, Vulnerability Assessment
Performance Testing Kiểm thử hiệu năng 1 test case Load Testing, Stress Testing

7. KẾT LUẬN VÀ PHƯƠNG PHÁP LUẬN TỔNG QUAN

Phương pháp luận Kiểm thử FastFoodOnline

Báo cáo này trình bày kết quả kiểm thử với phương pháp luận rõ ràng:

  1. Test Design: Mỗi test case kiểm tra 4-5 chức năng liên quan, đảm bảo coverage cao và hiệu quả
  2. Multiple Techniques: Kết hợp Black Box, Grey Box, và White Box Testing để phát hiện lỗi từ nhiều góc độ
  3. Structured Approach: Áp dụng V-Model với các giai đoạn kiểm thử tương ứng với phát triển
  4. Metrics-driven: Đo lường chất lượng bằng các metrics định lượng
  5. Continuous Improvement: Phân tích defects và cải thiện quy trình dựa trên dữ liệu thực tế
